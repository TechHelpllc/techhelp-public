1. What MyTechHelp Is Built For

MyTechHelp is a peer-to-peer help app designed to make remote tech support feel simple, safe, and human. It helps someone requesting help connect with a trusted helper who can guide them through a support session.

Purpose

  • Give requesters a simple way to ask for help from someone they trust.
  • Let helpers connect by entering a 6-digit help code or responding to a trusted-helper request.
  • Allow screen sharing only after the requester starts it, so the requester stays in control.
  • Keep support approachable for families, friends, and non-technical users.
MyTechHelp is built around real people helping real people — without giving the helper control of the requester’s device.

2. Built Around Trust, Security, and User Friendliness

Requester control

The requester decides when a session begins, when screen sharing starts, and when the session ends.

View-only help

Helpers can view the screen only after screen sharing is started by the requester. Helpers do not control the requester device.

Secure request flow

Help codes expire automatically. Trusted-helper requests can be accepted, declined, cancelled, or allowed to time out.

Clear history

Previous Requests, Request Queue, Help History, and Diagnostics help users understand what happened during support.

Privacy note: Screenshots in public materials should blur names, phone numbers, email addresses, account IDs, and private device content before publishing.

3. Requester Flow

Use the requester flow when you need help. You can create a secure help code, ask a trusted helper, start screen sharing, and review previous requests afterward.

Start from the Request Help tab

  1. Open MyTechHelp.
  2. Stay on the Request Help tab.
  3. Tap Get Help Now to create a secure help code, or open Trusted Helpers to ask someone already saved.

Request Help dashboard

Screenshot: Request Help dashboard Get Help Now, Trusted Helpers, Previous Requests, Settings

Start here to create a code, manage trusted helpers, or review previous requests.

Get Help Now / secure code

Screenshot: Get Help Now screen 6-digit help code and expiration timer

Use Get Help Now to create a 6-digit code. Codes expire automatically for security.

Use Trusted Helpers

  1. From the Request Help tab, open Trusted Helpers.
  2. Tap Add Trusted Helper to add someone you trust.
  3. Tap Ask for Help under a trusted helper to send a support request.
  4. You can cancel the request before it connects if you no longer need help.

Trusted Helpers

Screenshot: Trusted Helpers list Ask for Help and Remove buttons

Trusted Helpers lets you ask saved helpers for support without sharing a help code.

Previous Requests

Screenshot: Previous Requests Completed, Timed Out, Cancelled, Declined

Previous Requests shows what happened after you asked for help.

Start Screen Sharing

  1. After the session connects, tap Start Screen Share.
  2. When the Apple broadcast screen appears, choose the MyTechHelp broadcast extension.
  3. Tap Start Broadcast.
  4. After sharing begins, your helper can view your screen but cannot control it.
  5. Navigate to the app or screen where you need help.

During a Requester Session

Screen Sharing Active

Screenshot: Screen Sharing Active Call Helper and End Session options

End Session Confirmation

Screenshot: End Session confirmation Keep Open or End Session

Tap End Session, then confirm. Phone or FaceTime calls may continue separately and should be ended separately.

End or Recover a Requester Session

  • If the requester ends the session, MyTechHelp ends the support session and asks the broadcast extension to stop screen sharing.
  • If the helper ends the session, the requester app detects the ended session and stops screen sharing.
  • If the requester force-closes the app and later reopens it while a session is active, MyTechHelp can offer to rejoin or end the session.

Previous Request Statuses

Status Meaning
Completed A support session connected and ended normally.
Waiting A request is still active and waiting for a response.
Timed Out A trusted-helper request expired before it was accepted.
Cancelled The requester withdrew the request before it connected.
Declined The helper declined the request before a session started.

4. Helper Flow

Use the helper flow when someone asks you for help. Helpers can enter a help code, respond to trusted-helper requests, review the Request Queue, and review completed sessions.

Provide Help dashboard

  • Enter Help Code connects to a requester using their 6-digit code.
  • Request Queue shows pending, missed, timed-out, cancelled, and declined trusted-helper requests.
  • Help History shows completed support sessions.

Provide Help dashboard

Screenshot: Provide Help dashboard Enter Help Code, Request Queue, Help History

Helpers can enter a help code, review incoming requests, or open help history.

Enter Help Code

Screenshot: Enter Help Code screen 6-digit code keypad

Enter the requester’s 6-digit code only when the requester is expecting help.

Respond to Trusted-Helper Requests

  1. When a trusted-helper request arrives, review the requester name.
  2. Tap Accept if you are ready to help.
  3. Tap Decline if you cannot help.
  4. If the request is not answered in time, it appears as timed out or missed in Request Queue.

Request Queue

Screenshot: Request Queue Pending, missed, timed out, cancelled, declined

Request Queue is the helper inbox for trusted-helper requests that did not become completed sessions.

Help History

Screenshot: Help History Completed support sessions

Help History contains completed support sessions and basic session details.

During a Helper Session

  1. After accepting, the helper session opens.
  2. Wait for the requester to start screen sharing.
  3. Use Refresh Screen if the view appears stale.
  4. Use Call Requester if a support call contact is available.
  5. Use End Session if the support session should be ended for both users.

5. Settings, Features, and Pricing

Settings is where users manage account information, plan details, diagnostics, app preferences, support resources, guided walkthrough, and account deletion.

Pricing Model

  • Everyone receives a 30-day free trial.
  • After the 30-day free period, requester support sessions require Premium access.
  • Premium unlocks unlimited support sessions and future advanced features.
  • Helpers can still help someone else without a subscription unless they are requesting help themselves.

Pricing note: The Plan & Usage card shows whether the user is in trial, expired trial, or Premium status.

Diagnostics

  • Open Settings.
  • Tap Diagnostics.
  • Choose the relevant session log folder.
  • Tap Export Logs or Email Logs to Support.

Diagnostics exports logs that help troubleshoot support sessions. The current export includes session logs and app startup/lifecycle logs so support can see what happened before and during a session.

Delete Account & Data

Screenshot: Delete Account & Data Permanent account deletion warning and removal list

Account deletion is permanent. Review what will be removed or anonymized before confirming.

  • Account deletion is permanent and cannot be undone.
  • MyTechHelp attempts to remove account profile information, trusted-helper relationships, push tokens, pending requests, help codes, and session records connected to the account.
